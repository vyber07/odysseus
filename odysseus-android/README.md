## What's New in v2.1.0

- **Mobile Token Auth** — generate a token in Settings → Mobile App on the webapp, paste it in the app. No more cookie sessions — the `ody_…` bearer token works reliably across all networks.
- **Token paste on first run** — the setup screen now has an optional token field so you can connect in one step.
- **Bearer token persisted** — stored in DataStore, survives app restarts and background kills.
- **QR connect** — scan the QR generated by the webapp to fill in server URL + token automatically.

## Quick Setup

### Option A — Token (recommended)

1. In the webapp go to **Settings → Mobile App** → enter a name → **Generate Token**
2. Copy the `ody_…` token or note the QR
3. Install the APK → enter server URL → paste token → **Continue**

### Option B — Password only

1. Install the APK
2. Enter server URL (e.g. `http://YOUR_EC2_IP/`)
3. Tap **Test Connection** → **Continue to Login**
4. Enter your Odysseus username and password

## Connecting to Your Server

The Odysseus server must be publicly accessible:

```bash
# On your server
git clone https://github.com/vyber07/odysseus.git
cd odysseus
cp .env.example .env
# Set APP_BIND=0.0.0.0 and APP_PORT=80 (these are the defaults)
docker compose up -d --build
```

**If on AWS EC2** — open inbound port 80 in your Security Group:

| Type | Port | Source |
|------|------|--------|
| HTTP | 80   | 0.0.0.0/0 |
| SSH  | 22   | Your IP only |

## Mobile Token Details

Tokens are scoped API keys that the app sends as:

```
Authorization: Bearer ody_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

The `mobile_app` profile grants: `mobile · chat · todos:read/write · documents:read/write · email:read/draft · calendar:read/write · memory:read/write`

**Manage sessions** in the webapp under **Settings → Mobile App → Active Mobile Sessions** — revoke any token you don't recognise.

## Architecture

```
app/src/main/java/com/odysseus/wrapper/
├── core/
│   ├── NetworkClient.kt      # OkHttp + Retrofit + cookie jar + bearer token interceptor
│   └── UserPreferences.kt    # DataStore (server URL, login state, bearer token, dark mode)
├── data/
│   ├── remote/
│   │   ├── Models.kt         # All request/response data classes
│   │   └── ApiServices.kt    # All Retrofit service interfaces
│   └── repository/
│       └── Repositories.kt   # All repository classes
└── ui/
    ├── screens/
    │   ├── auth/             # FirstRunScreen (+ token paste), LoginScreen, AuthViewModel
    │   ├── chat/             # ChatScreen, ChatViewModel
    │   ├── notes/            # NotesScreen, NotesViewModel
    │   ├── tasks/            # TasksScreen, TasksViewModel
    │   ├── calendar/         # CalendarScreen, CalendarViewModel
    │   ├── email/            # EmailScreen, EmailViewModel
    │   ├── documents/        # DocumentsScreen, DocumentsViewModel
    │   ├── gallery/          # GalleryScreen, GalleryViewModel
    │   ├── research/         # ResearchScreen, ResearchViewModel
    │   ├── memory/           # MemoryScreen, MemoryViewModel
    │   └── settings/         # SettingsScreen
    ├── components/           # Shared composables (LoadingBox, EmptyBox, StatusChip)
    └── theme/                # Exact Odysseus webapp color palette (dark + light)
```

**Tech stack:** Kotlin · Jetpack Compose · Material 3 · Retrofit 2 · OkHttp · DataStore · Coil · Navigation Compose

## Building from Source

### Prerequisites

- Android Studio Hedgehog 2023.1.1+ (or newer)
- JDK 17 (bundled with Android Studio)
- Android SDK 34

### Steps

```bash
git clone https://github.com/vyber07/odysseus.git
cd odysseus/odysseus-android
```

Open in Android Studio → wait for Gradle sync → **Run ▶** on device or emulator.

No API keys, no `.env` file — the app is configured at runtime via the server URL and optional token.

## Auth

Two auth modes work in parallel:

| Mode | How | When used |
|---|---|---|
| **Bearer token** | `Authorization: Bearer ody_…` header on every request | When a mobile token is stored |
| **Cookie session** | `odysseus_session` cookie (same as browser) | After username/password login |

Both can be active simultaneously. The token is injected by an OkHttp interceptor.

## Contributing

PRs welcome. To add a feature from the Odysseus web API:

1. Add request/response models to `data/remote/Models.kt`
2. Add the Retrofit interface to `data/remote/ApiServices.kt`
3. Add a repository method to `data/repository/Repositories.kt`
4. Add a ViewModel + Screen under `ui/screens/`
5. Register the route in `MainActivity.kt`
